# 🔧 解决乱码问题

## 问题描述

你遇到的乱码问题是因为：
1. **Windows CMD 默认使用 GBK 编码**
2. **脚本文件使用 UTF-8 编码**
3. **两者编码不匹配导致中文显示乱码**

## ✅ 解决方案（3 种）

---

## 方案 1：使用 PowerShell 脚本（最推荐，无乱码）

### 步骤 1：使用新的脚本

我创建了 **PowerShell 版本**的脚本，完全避免乱码问题：

- **`一键打包_推荐.ps1`** - 打包脚本（推荐使用）
- **`一键启动_推荐.ps1`** - 启动脚本（推荐使用）

### 步骤 2：如何运行

**方法 A：右键运行（推荐）**

1. 找到 `一键打包_推荐.ps1` 文件
2. 右键 → "使用 PowerShell 运行"
3. 如果提示"无法加载脚本"，请看下面的方法 B

**方法 B：通过 PowerShell 运行**

1. 按 `Win + X`，选择 "Windows PowerShell (管理员)"
2. 进入项目目录：
   ```powershell
   cd 你的项目路径\hospital-schedule
   ```
3. 运行脚本：
   ```powershell
   .\一键打包_推荐.ps1
   ```

**如果提示"无法加载脚本"**：

1. 按 `Win + X`，选择 "Windows PowerShell (管理员)"
2. 运行以下命令允许脚本执行：
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. 输入 `Y` 确认
4. 重新运行脚本

## 方案 3：修改 CMD 编码（不推荐）

### 临时修改编码

1. 打开 CMD
2. 运行：
   ```cmd
   chcp 65001
   ```
3. 再运行旧的 `一键打包.bat` 脚本

**缺点**：
- 只对当前窗口有效
- 关闭窗口后需要重新设置
